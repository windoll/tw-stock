# Code Review — 台股即時報價 (`index.html`)

審查日期：2026-06-16　範圍：單一檔案 `index.html`（HTML + CSS + JS，約 760 行）

---

## 1. 架構概觀

純前端單檔應用，無建置流程、無後端，狀態全存在瀏覽器 `localStorage`，可直接部署 GitHub Pages。

**狀態（localStorage keys）**

| Key | 內容 |
|---|---|
| `tw_watchlist_v1` | 自選與交易紀錄 `[{code, lots:[{date,type,shares,price}]}]` |
| `tw_div_cache_v1` | 上次同步的股利資料 |
| `tw_view_v1` / `tw_priv_v1` / `tw_hcols` | 檢視模式 / 隱私開關 / 隱藏欄位 |
| `tw_stocklist` | 股票代號↔名稱清單（快取 7 天） |

**資料來源**

| 用途 | 來源 | 取得方式 |
|---|---|---|
| 即時報價、指數 | TWSE `mis.twse.com.tw` | 經公用 CORS proxy 競速（瀏覽器無法直連） |
| 股利、股價、股票清單 | FinMind API | 瀏覽器直連（支援 CORS） |

**模組分區（以註解分段）**：交易紀錄推導 → 股票清單 → 報價抓取 → 計算 → 渲染（表格/卡片/總計/佔比/指數/提醒）→ 彈窗（交易/走勢圖）→ 股利同步與渲染 → 事件與初始化。

---

## 2. 優點

- **零依賴、零建置**，單檔即部署，維護成本低。
- **容錯良好**：proxy 以 `Promise.any` 競速 + `AbortController` 逾時取消；FinMind 失敗以 try/catch 降級，不影響其他功能。
- **資料相容**：`load()` 會把舊格式 `{code,qty,cost}` 自動轉成交易紀錄。
- **計算正確性**：股利以 `sharesAt(除息日)` 取當下持股計算，跨年增減持股也正確。
- **資料保全意識**：提供匯出／匯入，緩解 localStorage 易失問題。
- **體驗**：隱私模糊、手機自動精簡欄位、背景分頁暫停更新（省效能與 API 額度）。

## 3. 問題與風險（依嚴重度）

| # | 嚴重度 | 位置／主題 | 說明 | 建議 |
|---|---|---|---|---|
| 1 | 中 | 持久層 | `localStorage` 是唯一儲存，清快取／換裝置即遺失。 | 已有匯出；可加「距上次備份過久」提醒或自動下載備份。 |
| 2 | 中 | 外部相依 | 報價依賴公用 CORS proxy，股利依賴 FinMind 免費額度（匿名約 300 次/時）。第三方不穩時功能受影響。 | 可加選填 FinMind token（提升額度、綁個人帳號）；proxy 已多來源競速。 |
| 3 | 低 | 安全性 | 多處用字串模板組 `innerHTML`，其中股票名稱（`nameMap`，來自 API）直接插入。來源可信且為本機個人工具，風險低，但非零。 | 對插入的外部字串做簡單跳脫（`&<>"`）。 |
| 4 | 低 | 一致性 | 「殖利率」用近一年(TTM)，「含息報酬」用累計實際領取，兩者口徑不同（刻意設計），但 UI 未標註定義。 | 在欄位旁加 tooltip 或說明，避免誤解。 |
| 5 | 低 | 效能 | 每次更新重建整個表格 `innerHTML`，並重繪佔比圖／提醒。股票數量大時有微小重繪成本。 | 實務上無感；若清單變很大再考慮差異更新。 |
| 6 | 低 | 測試 | 無自動化測試，驗證靠手動／瀏覽器主控台。 | 可抽出純函式（`curShares`/`avgCost`/`sharesAt`/`cashDivReceived`）做單元測試。 |
| 7 | 資訊 | 結構 | 全域變數集中於 script 頂層（`watch`/`divData`/`nameMap`/`view`/`timer`…）。單檔可接受，但無命名空間封裝。 | 若持續成長可包進單一物件或 IIFE 模組。 |

## 4. 本次整理與修正（已完成）

- 移除死碼：`input.hold` 相關 CSS（表格內嵌輸入欄已改為交易紀錄彈窗，樣式不再使用）。
- 確認無孤立函式（`fetchJsonProxy`、`setHold`、`rocSlash`、`sumFields` 等舊版殘留皆已不存在）。
- **#3 安全性**：新增 `esc()` HTML 跳脫，套用於所有插入 `innerHTML` 的外部字串（股票名稱、指數名稱、datalist 選項）。
- **#4 一致性**：「殖利率」欄與「含息損益」加上 `title` 說明各自定義，避免口徑誤解。
- **#7 結構**：整段 JS 包進 IIFE，全域變數不再外洩到 `window`；僅刻意暴露 `window.StockCalc`（純計算函式）供測試。
- **#6 測試**：新增 `tests.html`，以隱藏 iframe 載入 index.html 直接測試 `window.StockCalc`（curShares／avgCost／sharesAt／cashDivReceived），不複製程式碼、無漂移。全部案例通過。
- **#1、#2**：依需求暫不處理（持久層、第三方相依）。
- **#5**：效能差異更新依結論「實務無感」不實作，避免過度工程。

> 採保守原則（surgical changes）：未對運作正常的邏輯做破壞性重構，維持單檔部署的簡潔與穩定。

### 如何跑測試
與 `index.html` 放同一資料夾，用瀏覽器開啟 `tests.html`（或部署後開 `.../tests.html`）即可看到通過/失敗結果。

## 5. 後續建議（不破壞現狀、可逐步做）

1. **資料保全**：自動備份提醒，或變更後自動下載 JSON。
2. **FinMind token 欄位**：選填，提升額度並與他人分流。
3. **純函式單元測試**：對金額／持股計算建立測試，鎖住商業邏輯。
4. **外部字串跳脫**：插入 `innerHTML` 的股名做 escape。
5. **版本號／CHANGELOG**：檔頭加版本註解，利於追蹤。

---

## 6. 結論

以「個人用、純前端、單檔部署」的定位來看，程式碼結構清晰、容錯與計算正確性良好，目前**可維持單檔不需大改**。主要風險集中在「資料只存本機」與「依賴第三方資料源」兩點，皆有緩解手段，屬可接受範圍。
